Design principles

Four rules the optimizer never breaks.

Joule sits between your application and any provider. These constraints keep it honest about what it changes and why.

01

Measure before optimizing

Energy should be observable. Every inference exposes latency, input and output tokens, estimated energy in joules, electricity in Wh, CO₂, and cost — before a single optimization runs.

⊳ measured  1,820 J · 0.51 Wh · 0.23 g CO₂ · $0.0034
02

Quality first

Reducing energy must never meaningfully reduce answer quality. Every pass is measurable and reversible, so you can see exactly what an optimization traded — and undo it.

03

Model agnostic

The pipeline stays independent of the model vendor. Point it at OpenAI, Anthropic, Gemini, Ollama, LM Studio, vLLM, or llama.cpp — the optimization logic doesn't change.

openai · anthropic · gemini · ollama · vllm · llama.cpp
04

Transparent

Nothing happens invisibly. Joule explains every optimization it applies and reports the estimated energy it saved, per request — in headers and in a human-readable summary.

Architecture

One hop between your app and the provider.

Applications keep using familiar APIs. Joule transparently measures, optimizes, caches, and routes every request that passes through.

Source
Application
JouleOpenAI-compatible proxy
Proxy

Receives requests (OpenAI-compatible) with timeouts, retries, and per-provider circuit breaking.

Optimizer

Removes duplicated context, compresses prompts, trims verbosity, recommends output limits.

Cache

Avoids repeated inference — exact-match and semantic (embedding-similarity).

Router

Selects a provider/model by support, estimated energy, grid carbon intensity (live-feed capable), or task complexity — static · model · greenest · carbon · complexity.

Estimator

Estimates energy from model, hardware, token count, and published benchmarks.

Metrics

Exports joules, CO₂, cost, and savings to Prometheus, with a SQLite request log — OpenTelemetry traces next.

Destination
LLM Provider

What Joule is not: a foundation model, an LLM framework, a prompt-engineering assistant, a model-serving engine, or a benchmark suite. It is infrastructure that makes the AI systems you already run more efficient.
