How many joules did that response cost?

Joule is an OpenAI-compatible proxy that measures the energy, CO₂, and cost of every LLM request — then optimizes the prompt and routes to greener models. In Rust.

the pipeline

Measure. Optimize. Route.

Every request flows through three stages. Each one is a plugin, each one explains itself, and nothing happens invisibly.

01 — measure

See the joules

Token-accurate accounting turns every request into joules, watt-hours, CO₂, and cost — calibrated to published benchmarks.

  • Prometheus /metrics
  • SQLite request log
  • x-joule-* headers
02 — optimize

Trim the prompt

Composable passes strip redundant tokens before inference — and report exactly what they removed and saved.

  • lite · full · ultra
  • dedup · strip filler
  • output limits
03 — route

Pick the greenest

Send each request to the provider and model that fit — by support, or by lowest estimated energy.

  • static · model · greenest
  • OpenAI / Anthropic / Gemini
  • local Ollama, vLLM…
show, don't tell

Point your client at Joule. Read the receipts.

Same OpenAI API you already use. The energy comes back on every response — these are real outputs, not mockups.

request
# talk to Joule exactly like OpenAI
curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o-mini",
       "messages":[{"role":"user",
       "content":"Hello!"}]}' -i

joule optimize

A prompt improver that speaks in joules.

Run it standalone or inline on the proxy. Lossless by default — whitespace, duplicate messages, repeated lines, filler. Every pass is reversible and reported. Nothing about your prompt changes silently.

See the passes
echo "…" | joule optimize --level full
Optimization Summary (full)
  ✓ normalized whitespace
  ✓ collapsed repeated lines
  ✓ removed filler phrases
  Prompt tokens: 49 → 23  (−26, 53%)

Prompt energy (gpt-4o):
  29.4 J → 13.8 J  (saved 15.6 J)
model-agnostic

One API. Any model. Translated for you.

Clients always speak OpenAI. Joule translates to each vendor's wire format and back — so switching providers is a config line, not a rewrite.

Routers

Choose the selection policy. Swapping it is a config change.

static

Always the default provider — a transparent proxy.

model

Route by which provider supports the requested model.

greenest

Pick the lowest-estimated-energy model among candidates.

honest about the work

Shipped in v0.1.0 — more on the way.

Joule measures, optimizes, and routes today. The biggest unbuilt lever — caching a repeat question down to near-zero energy — is what's next.

no API key needed

See it work in 30 seconds.

Point Joule at a local Ollama server and watch the energy headers come back — no cloud, no key, no cost.

  1. Build the single binary with cargo build --release.
  2. Run it pointed at any OpenAI-compatible upstream.
  3. Watch x-joule-* headers on every response.
terminal
# 1 · local model, no key
ollama serve & ollama pull llama3.2

# 2 · point Joule at it
joule serve --upstream http://localhost:11434

# 3 · send a request, read the energy
curl localhost:8080/v1/chat/completions \
  -d '{"model":"llama3.2",
   "messages":[{"role":"user",
   "content":"Say hi"}]}' -i | grep x-joule
